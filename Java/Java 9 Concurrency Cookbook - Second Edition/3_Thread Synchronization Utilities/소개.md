# 소개

2 장, **기본 스레드 동기화**에서 동기화의 개념과 중요 섹션을 배웠습니다. 기본적으로 둘 이상의 동시 작업이 리소스를 공유 할 때 동기화에 대해 이야기합니다 (예 : 개체 또는 개체의 속성). 이 공유 리소스에 액세스하는 코드 블록을 중요 섹션이라고합니다.

적절한 메커니즘을 사용하지 않으면 잘못된 결과, 데이터 불일치 또는 오류 조건이 발생할 수 있습니다. 따라서 이러한 문제를 피하기 위해 Java 언어에서 제공하는 동기화 메커니즘 중 하나를 채택해야합니다.

2 장, **기본 스레드 동기화**에서는 다음과 같은 기본 동기화 메커니즘에 대해 설명했습니다.

* `synchronized` 키워드
* Lock 인터페이스 및 구현 클래스 : `ReentrantLock`, `ReentrantReadWriteLock.ReadLock` 및 `ReentrantReadWriteLock.WriteLock`
* `StampedLock` 클래스

이 장에서는 고급 메커니즘을 사용하여 여러 스레드를 동기화하는 방법을 배웁니다. 이러한 고급 메커니즘은 다음과 같습니다.

* **세마포어(Semaphores)** : 세마포어는 하나 이상의 공유 리소스에 대한 액세스를 제어하는 ​​카운터입니다. 이 메커니즘은 동시 프로그래밍의 기본 도구 중 하나이며 대부분의 프로그래밍 언어에서 제공됩니다.
* **CountDownLatch** : `CountDownLatch` 클래스는 스레드가 여러 작업의 완료를 기다릴 수 있도록 Java 언어에서 제공하는 메커니즘입니다.
* **CyclicBarrier** : `CyclicBarrier` 클래스는 공통 지점에서 여러 스레드의 동기화를 허용하는 Java 언어에서 제공하는 또 다른 메커니즘입니다.
* **Phaser** : `Phaser` 클래스는 여러 단계로 나누어 진 동시 작업의 실행을 제어하는 ​​Java 언어에서 제공하는 또 다른 메커니즘입니다. 모든 스레드는 다음 단계를 계속하기 전에 한 단계를 완료해야합니다.
* **Exchanger** : `Exchanger` 클래스는 두 스레드 간의 데이터 교환 지점을 제공하는 Java 언어에서 제공하는 또 다른 메커니즘입니다.
* **CompletableFuture** : `CompletableFuture` 클래스는 하나 이상의 작업이 향후 비동기 방식으로 명시 적으로 완료 될 다른 작업의 완료를 기다릴 수있는 메커니즘을 제공합니다. 이 클래스는 Java 8에서 도입되었으며 Java 9에서 새로운 메소드를 도입했습니다.

세마포는 문제의 중요한 섹션을 보호하는 데 사용할 수있는 일반적인 동기화 메커니즘입니다. 다른 메커니즘은 앞에서 설명한대로 특정 기능이있는 응용 프로그램에서 사용되는 것으로 생각됩니다. 애플리케이션의 특성에 따라 적절한 메커니즘을 선택해야합니다.
